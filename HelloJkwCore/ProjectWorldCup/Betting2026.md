2026 월드컵 내기 사이트 개발 방향

# 기존 대회와 2026년 대회의 큰 차이점
- (기존) 32개국 출전 -> (변경) 48개국 출전으로 변경
- (기존) 4개팀 8개 조 -> (변경) 4개 팀 12개 조
- (기존) 조별리그 후 각 조 1,2위 16강 -> (변경) 조별 리그 후 각 조 1,2위 32강 직행, 조3위 중 8팀 32강 진출
- (기존) 토너먼트 16강 부터 -> (변경) 토너먼트 32강 부터


# 내기 사이트에서 같은 점
- 조별 리그 1,2위 맞추는 것은 동일
- 16강 진출 팀 중 8강 진출팀 맞추는 것 동일
- 8강 진출 팀 중 1,2,3,4 위 맞추는 것 동일

# 내기 사이트에서 달라지는 점
- 32강 진출팀 중 16강 진출팀 맞추기 추가.
- (기존) 3번(조별리그, 16강, 8강)의 내기 -> (변경) 4번(조별리그, 32강, 16강, 8강)의 내기

# 개발 방향
- 'Wc2026' 폴더 만들어서 그 하위에 page 파일 생성.
- 'Wc2022'와 비슷한 형태로 만들 것.
- 크게 GroupStage,Round32, Round16, Final 로 구분할 것.
- 기존 사용하던 컴퍼넌트를 재사용하는 것은 허용. 대신 url에 2022가 있다면 url은 2026으로 변경할 것. (Wc2018, Wc2022 폴더에 있는 파일은 재사용하지 말 것.)
- 메뉴 부터 변경 할 것. top-down 방식으로 개발 할 것.
- 조별리그 정보 표시에서 GetGroupOverview 함수는 사용하지 말고, GetStandingDataAsync　함수만　사용하도록　변경．
- fifa관련 api는 이미 손 본 상태. 더 이상 건드리지 말 것.
- 중간중간 git 커밋 할 것. 커밋하기 전에 나에게 변경사항 커밋하겠다고 물어볼 것.
