# 따님 식단 짜기 스펙

## 1. 목표

2살 아이의 식단을 보호자가 직접 등록하고 관리하는 서비스다. 등록된 메뉴와 재료 정보를 바탕으로 냉장고 재료로 만들 수 있는 메뉴를 찾고, 주간 장보기와 식단 추천을 함께 제공한다. 캘린더에는 실제로 먹인 식단을 날짜/끼니별로 기록한다.

## 2. 확정 원칙

- 사용자는 보호자이고, 식단 대상은 2살 아이다.
- 식단 데이터는 외부에서 가져오지 않고 사용자가 직접 등록한 메뉴만 사용한다.
- `식단`은 한 끼 전체가 아니라 밥, 국, 반찬, 간식처럼 조합 가능한 개별 메뉴다.
- 한 끼는 여러 메뉴 조합으로 구성될 수 있다.
- 모든 재료는 재료명, 수량, 단위를 가진다.
- 단위는 제한하지 않는다. 예: `g`, `ml`, `개`, `모`, `팩`, `큰술`, `작은술`, `컵`, `조각`, `줌`.
- 같은 재료명과 같은 단위끼리는 합산한다.
- 서로 다른 단위 환산은 아직 결정하지 않는다.
- 양념, 육수, 조미료 같은 기본 재료는 메뉴 재료에는 등록할 수 있지만 장보기/부족 재료 계산에서는 제외한다.
- 냉장고 재료는 재료명과 수량을 입력한다.
- 남은 재료는 자동으로 냉장고에 이월하지 않고, 사용자가 직접 냉장고 재료로 기록한다.
- 최근 먹인 메뉴는 참고 정보로만 보여주며 추천 우선순위를 자동으로 낮추지 않는다.
- 아이의 끼니별 반응은 기록하지 않는다.

## 3. 핵심 데이터

### 3.1 메뉴

사용자가 등록한 개별 음식이다. 예: `소고기무국`, `애호박볶음`, `계란찜`, `닭고기야채죽`.

필드 후보:

- 이름
- 유형: 밥, 국, 반찬, 간식 등
- 재료 목록
- 아이가 잘 먹는 정도
- 조리 난이도
- 조리 시간
- 아침 추천 적합 여부
- 미리 만들어둘 수 있는지 여부
- 메모

### 3.2 메뉴 재료

필드 후보:

- 재료명
- 수량
- 단위
- 필수 여부
- 기본 재료 여부
- 대체 가능한 재료 목록

기본 재료는 장보기 목록과 부족 재료 계산에서 제외한다.

### 3.3 주요 재료

장보기와 추천에서 재사용을 우선 고려할 재료다. 예: `소고기`, `닭고기`, `두부`, `애호박`, `양배추`.

주요 재료는 사용자가 직접 표시하지 않고 시스템이 자동 판단한다. 판단 기준 후보는 다음과 같다.

- 여러 메뉴에서 반복 사용되는 재료
- 최소 구매 단위가 커서 남기 쉬운 재료
- 가격 또는 양 때문에 한 번 사면 나눠 쓰기 좋은 재료
- 기본 재료가 아닌 재료

### 3.4 구매 단위

실제 장보기에서 재료를 최소로 사야 하는 단위다.

예:

- 소고기 최소 300g
- 두부 최소 1모
- 애호박 최소 1개
- 계란 최소 10개

요구사항:

- 재료별 최소 구매 단위를 등록할 수 있어야 한다.
- 메뉴별 필요량 합계와 최소 구매 단위를 비교해 실제 구매 권장량을 계산한다.
- 사용자는 추천 구매 수량을 직접 수정할 수 있다.
- 수정된 구매 수량은 장보기 목록과 예상 잔여량에 반영한다.

### 3.5 아이 제한 정보

등록 정보:

- 알레르기 재료
- 싫어하는 재료
- 아직 먹이면 안 되는 재료
- 잘 먹는 음식 또는 선호 메뉴

추천 반영:

- 알레르기 재료가 포함된 메뉴는 추천하지 않는다.
- 아직 먹이면 안 되는 재료가 포함된 메뉴는 추천하지 않는다.
- 싫어하는 재료가 포함된 메뉴는 제외하지 않고 낮은 우선순위로 표시한다.
- 잘 먹는 음식은 추천 우선순위를 높인다.

### 3.6 대체 재료

메뉴 재료에는 대체 가능한 재료를 등록할 수 있어야 한다. 예: 애호박 대신 브로콜리, 소고기 대신 닭고기, 두부 대신 계란.

대체 재료는 다음 상황에서 사용한다.

- 냉장고에 원래 재료는 없지만 대체 재료가 있는 경우
- 장보기 추천에서 특정 재료를 사면 더 많은 메뉴를 만들 수 있는 경우
- 알레르기나 아직 먹이면 안 되는 재료를 피해야 하는 경우

### 3.7 캘린더 기록

날짜와 끼니별로 실제로 먹인 메뉴 조합을 기록한다. 추천된 식단과 실제로 먹인 식단은 다를 수 있다.

필드 후보:

- 날짜
- 끼니: 아침, 점심, 저녁 등
- 메뉴 목록
- 직접 입력 메모

## 4. 주요 기능

### 4.1 메뉴 등록/관리

사용자는 메뉴 이름, 유형, 재료, 수량, 단위, 조리 정보, 선호 정보를 등록/수정/삭제할 수 있어야 한다.

### 4.2 냉장고 재료 기반 메뉴 찾기

사용자는 현재 냉장고에 있는 재료와 수량을 입력한다.

시스템은 등록된 메뉴 중 다음을 보여준다.

- 지금 바로 만들 수 있는 메뉴
- 부족한 재료가 1~2개인 메뉴
- 메뉴별 부족 재료 목록

정렬 기준 후보:

- 부족한 재료가 적은 순서
- 부족한 재료를 사면 다른 메뉴에도 재사용할 수 있는 순서
- 아이가 잘 먹는 순서

최근 먹인 메뉴 여부는 정렬 기준으로 사용하지 않고 참고 정보로만 보여준다.

### 4.3 주간 장보기 기반 추천

등록된 메뉴 목록을 기반으로 주간 메뉴 조합과 장보기 목록을 함께 추천한다.

추천 대상:

- 주중: 아침, 저녁
- 주말: 아침, 점심, 저녁

끼니별 메뉴 수:

- 아침: 간단한 메뉴 위주로 1~2개
- 주말 점심: 1~2개 또는 2~3개
- 저녁: 2~3개

추천 기준:

- 같은 주요 재료를 여러 메뉴에서 재사용할 수 있는가
- 실제 구매 단위 대비 남는 재료가 적은가
- 한 주 동안 메뉴가 과도하게 반복되지 않는가
- 아이가 먹을 수 있는 메뉴인가
- 장보기 목록이 과도하게 늘어나지 않는가
- 아침에는 빠르고 편한 메뉴인가
- 주말에는 조리 시간이 긴 메뉴도 배치 가능한가
- 싫어하는 재료가 포함된 메뉴는 낮은 우선순위로 밀리는가
- 한 끼와 주간 전체에서 영양 균형이 맞는가

추천 결과:

- 요일별/끼니별 추천 메뉴
- 메뉴별 필요한 재료
- 카테고리별 통합 장보기 목록
- 재료별 총 필요량
- 실제 구매 권장량
- 예상 잔여량
- 잔여 재료로 만들 수 있는 추가 후보 메뉴

장보기 목록은 `육류`, `채소`, `유제품`, `곡류`, `과일`, `기타` 같은 카테고리로 묶어 보여준다. 추천안은 사용자가 확정해야 캘린더에 등록한다.

### 4.4 영양 균형 추천

영양 역할은 사용자가 메뉴에 직접 등록하지 않고, 시스템이 재료를 보고 자동 판단한다.

초기 분류:

- 탄수화물: 밥, 죽, 면, 감자
- 단백질: 소고기, 닭고기, 계란, 두부, 생선
- 채소: 애호박, 당근, 브로콜리, 양배추
- 과일/간식: 바나나, 사과, 요거트

추천 규칙:

- 한 끼에 단백질과 채소가 포함되도록 우선 추천한다.
- 아침은 간단하되 단백질 또는 채소가 빠지지 않도록 한다.
- 저녁은 2~3개 메뉴 조합 안에서 탄수화물, 단백질, 채소 균형을 맞춘다.
- 주간 전체에서 특정 영양 역할이 지나치게 부족하지 않도록 한다.

### 4.5 캘린더 관리

사용자는 캘린더에서 날짜와 끼니를 선택해 실제로 먹인 메뉴를 기록한다.

필요 기능:

- 날짜별 식단 보기
- 끼니별 메뉴 추가/삭제
- 확정한 주간 추천 식단을 캘린더에 반영
- 등록된 메뉴를 다른 날짜/끼니로 복사
- 과거에 먹인 메뉴 확인

캘린더 기록은 추천 화면에서 최근 식단 참고 정보로 사용할 수 있지만 추천 우선순위를 자동으로 낮추는 데 쓰지 않는다.

## 5. 구현 우선순위

1. 메뉴/재료 등록과 수정
2. 냉장고 재료 입력 및 만들 수 있는 메뉴 검색
3. 부족 재료 1~2개 메뉴 추천
4. 주간 식단 추천과 장보기 목록 생성
5. 최소 구매 단위와 예상 잔여량 계산
6. 캘린더 기반 실제 식단 기록
7. 아이 제한 정보와 대체 재료 반영
8. 영양 균형 기반 추천 고도화

## 6. 미결정 사항

1. 서로 다른 단위를 같은 재료로 환산할 것인가?
2. 대체 재료는 모든 메뉴에 공통 적용할 것인가, 특정 메뉴 안에서만 적용할 것인가?
3. 대체 재료를 사용한 추천은 원래 메뉴명으로 보여줄 것인가, 변형 메뉴명으로 보여줄 것인가?
4. 주요 재료 자동 판단 기준의 우선순위는 무엇인가?
5. 자동 판단된 주요 재료를 사용자가 제외하거나 고정할 수 있어야 하는가?
6. 영양 자동 분류 결과를 사용자가 수정할 수 있어야 하는가?
7. 한 재료가 여러 영양 역할을 가질 수 있어야 하는가?
8. 냉장고 재료와 장보기 재료 카테고리는 자동으로 붙일 것인가, 사용자가 직접 정할 것인가?
9. 추천안 확정 후 캘린더에 등록되는 것은 계획 식단인가, 실제로 먹인 식단인가?
